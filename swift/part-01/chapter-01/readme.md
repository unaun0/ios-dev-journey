## 1.3 Базовые понятия

**Хранилище данных** - это виртуальный объект со следующими свойствами:
 - записанное значение;
 - идентификатор (имя);
 - тип информации, для хранения которой предназначено хранилище (числовой, строковый и др.).

Весь **процесс программирования** заключается в том, чтобы:
- создавать (объявлять) объекты, 
- задавать (инициализировать) им значения, 
- получать эти значения из памяти,
- производить с ними операции.

**Объявление** — это создание нового объекта (хранилища данных). 

**Инициализация** — это присвоение значения объекту.

**Любая программа** - это набор выражений (набор команд), понятных компьютеру. 

**Выражение** - команда, выполняющая одну или несколько операций. Выражение может состоять из множества операторов и операндов.

**Оператор** - это минимальная независимая функциональная единица, выполняющая определенную операцию.

**Операнд** - это значение, с которым оператор производит операцию.

Все зарезервированные языком программирования наборы символов называются **ключевыми словами**.

## 1.4 Введение в операторы

- **Простые операторы**, выполняющие операции со значениями (операндами) - их состав входят унарные и бинарные операторы.

- **Унарные операторы** выполняют операцию с одним операндом; могут находится перед операндом (префиксные) или после него (постфиксные).

- **Бинарные операторы** - выполняют операцию с двумя операндами (оператор, который располагается между операндами, называется инфиксным).

- **Структурные операторы** - влияют на ход выполнения программы.

## 1.5 Оператор инициализации

**Оператор инициализации (присваивания) (=)** - это бинарный оператор; используется в типовом выражении `a = b`, присваивая хранилищу данных с именем `a` значение хранилища данных с именем `b`.

## 1.6 Переменне и константы

Всего в Swift выделяют *два вида хранилищ данных*: 
- **переменные**, объявляемые с помощью ключевого слова `var`; 
- **константы**, объявляемые с помощью ключевого слова `let`.

Любое хранилище, неважно, какого вида, имеет *три важнейших свойства*:
1) **имя**: уникальное название, по которому можно обратиться к хранилищу;
2) **тип данных**: определяет множество допустимых значений и множество операций, которые можно выполнять над этими значениями;
3) **значение**: текущие данные, находящиеся в хранилище.

### Переменные

**Переменная** — это хранилище данных; именованная область памяти, значение которой может быть изменено в процессе выполнения программы.

```
var имя: Тип = значение
```

Оператор `var` используется *единожды* для каждой переменной только при ее *объявлении*.

```
var age: Int = 25
age = 26  // значение изменено
```

### Константы

**Константа** — это хранилище данных; именованная область памяти, начение которой не может быть изменено после инициализации.

```
let имя: Тип = значение
```

```
let pi = 3.14159
pi = 3.14  // Ошибка: cannot assign to value: 'pi' is a 'let' constant
```

### Объявление нескольких параметров в одном выражении

При объявлении нескольких параметров можно использовать один оператор — `var` или `let` и через запятую попарно указать их имена и инициализируемые значения.

```
var fruit1 = "Apple", fruit2 = "Banana"
var cost1 = 10, cost2 = 5
```

## 1.7 Инициализация копированием

При инициализации переменной или константы в правой части можно указывать не только литеральное значение, но и имя другого параметра.

```
let myAge = 40
var yourAge = myAge // yourAge = 40
```

При инициализации одного параметра значением другого создаются **две независимые области памяти**. Изменение одного параметра не **влияет на другой.

```
let myAge = 40
var yourAge = myAge

yourAge = 25
// myAge = 40, yourAge = 25 — значения независимы
```

Тип данных параметра, который обеспечивает передачу значения копированием, называется `value type` (значимый тип, или тип-значение).

**Value type (тип-значение)** - тип данных, при котором при присваивании или передаче в функцию создается копия значения. Каждый экземпляр хранит собственную независимую копию данных.

**Reference type (ссылочный тип)** - тип данных, при котором при присваивании или передаче в функцию создается копия ссылки на объект, а не сам объект. Несколько экземпляров могут ссылаться на один и тот же объект в памяти.

| Характеристика         | Value Type                                               | Reference Type                     |
|------------------------|----------------------------------------------------------|------------------------------------|
| **Механизм**           | Копируется само значение                                 | Копируется ссылка на объект        |
| **Независимость**      | Экземпляры независимы                                    | Изменение через любой экземпляр изменяет общий объект |
| **Хранение**           | Обычно в стеке (stack)                                   | В куче (heap)                      |
| **Управление памятью** | Автоматическое, без подсчета ссылок                      | ARC (Automatic Reference Counting) |
| **Примеры**            | `struct`, `enum`, `Int`, `String`, `Array`, `Dictionary` | `class`, `actor`, `closure`        |

## 1.8 Правила именования переменных и констант

При написании программ на Swift вы можете создавать параметры с любыми именами. При этом существуют правила, которых необходимо придерживаться:

1. Имя переменных и констант пишутся в *camelСase*.

При именовании используются латинские символы, без подчеркиваний, тире, математических формул и т. д. Каждое значимое слово, кроме первого, начинается с заглавной буквы.

Пример хорошего названия: userData, messageFromServer, highScoreOfAllUsers.


2. Имена параметров должны быть осмысленными и раскрывать их реальное предназначение. При этом ясность должна преобладать над краткостью.

## 1.10 Область видимости (scope)

**Область видимости** (scope) - границы, определяющие, где объект (переменная, константа, функция) может быть использован.

| Тип                   | Определение                                | Доступность             |
|-----------------------|--------------------------------------------|-------------------------|
| **Глобальный объект** | Объект, объявленный в корне программы      | В любой точке программы |
| **Локальный объект**  | Объект, объявленный внутри другого объекта | Только внутри родительского объекта |

**Правила**:

1. Доступ к локальным объектам извне осуществляется через родительский объект:

```
Friend.name
```

2. Одинаковые имена допустимы в разных областях видимости:

```
User.name    // один объект
Friend.name  // другой объект с тем же именем, но в другой области
```

3. Время жизни локального объекта равно времени жизни родительского объекта

**Область видимости** — механизм управления доступом, предотвращающий:
- Непреднамеренное изменение объектов из недопустимых мест
- Конфликты имен

## 1.12 Точка с запятой

Точка с запятой (;) в Swift **не требуется** в конце строки. Строка считается завершенной **по символу переноса**.

**Исключение**: точка с запятой нужна, когда в одной строке записано несколько выражений.

```
// одна строка — одно выражение (точка с запятой не нужна)
var number = 18

// одна строка — несколько выражений (точка с запятой обязательна)
number = 55; let userName = "Alex"
```

## 1.13 Отладочная консоль и функция print(_:)

### Консоль

**Консоль** — это интерфейс командной строки, в котором отображаются текстовые данные.

Одной из составных частей Xcode также является отладочная консоль. Ее основная функция — вывод отладочной текстовой информации (например, сообщений об ошибках), а также текстовых данных, указанных вами в процессе разработки программы.

Отладочная консоль выводит разработчику или пользователю необходимую информацию и используется в основном в двух случаях: для взаимодействия с пользователем приложения или для поиска ошибок на этапе отладки.

**Отладка** — это этап разработки программы, на котором обнаруживают и устраняют ошибки.

### Вывод текстовой информации

При работе в Xcode Playground значения переменных и констант отображаются в области результатов.

В реальных Xcode-проектах при создании приложений никакой области результатов нет, интерфейс среды разработки приложений отличается от интерфейса Xcode Playground.
Существует иной способ вывода информации, в том числе значений переменных. Будь то Playground или полноценное приложение для iOS или macOS, данные можно отобразить на отладочной консоли.

Вывод на консоль осуществляется с помощью функции `print(_:)`.

```
print("Hello world!")
print(number)
print(number, userName)
print("My name is \(userName)")
```

Вывод в консоли:
```
Hello world!
55
55 Alex
My name is Alex
```

**Функция** — это именованный фрагмент программного кода, к которому можно многократно обращаться. 

Процесс обращения к функции называется **вызовом функции**. Для вызова функции необходимо написать ее имя и конструкцию из скобок, например `myFunction()`.

Функции позволяют избежать дублирования кода: они группируют часто используемый код с целью многократного обращения к нему по имени.

Некоторые функции построены так, что при их вызове нужно передать входные данные, которые будут использованы функцией внутри нее - такие данные называются **аргументами функции**.

Не стоит путать понятия аргумент функции и входной параметр:

**Аргумент функции** — это то, что передается в функцию в скобках при ее вызове.
**Входной параметр** — это то, что используется внутри функции.

Каждая функция имеет **сигнатуру**, то есть *краткое описание*. Сигнатура содержит информацию об имени функции, ее аргументах и возвращаемом функцией значении. Может быть в одной из трех форм:
1. Полная, с указанием типов данных:
    ```
    someFunction(a:String, b: String) -> String
    ```
2. Стандартная, без указания типов данных:
    ```
    someFunction(a: b:)
    ```
3. Краткая, с указанием только имени функции:
    ```
    someFunction
    ```
